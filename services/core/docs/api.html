Формат запроса

Запрос состоит из следующих элементов:

  1. Заголовок Authorization (Значением данного заголовка должен быть токен приложения)
  2. Тело запроса в формате JSON

Формат ответа

Ответ на каждый запрос приходит в формате JSON. Набор полей в ответе зависит от метода, который был вызван. Перед тем, как начать парсить ответ, проверьте статусный код ответа. Если он равен 200, то это значит, что запрос прошел успешно. Если же он не равен 200, то при обработке запроса произошла ошибка. Что бы понять, что пошло не так, сравните код ошибки с кодом ошибок API

Коды ошибок

Коды ошибок API
Код ошибки Описание
400 Неверный формат запроса.
401 Неверный токен приложения.
403 Неверный ID пациента

Методы

/api/predict

Используйте данный метод, что бы получить вероятность заболевания.

Отправьте симптомы в теле POST запроса.

Необходимые JSON поля
Имя поля Тип поля Возможные значения Описание
patient_id int 0-32767 ID пациента
fever int 0-10 Температура
cough int 0-5 Кашель
shortness_of_breath int 0-4 Одышка
head_ache int 0-5 Головная боль
sore_throat int 0-5 Боль горла
additional_factor int 0-2
